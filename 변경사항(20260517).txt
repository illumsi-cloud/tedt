# 충주시 초·중학교 통학구역 및 중학구 검색 서비스 개발 변경 이력서

## 문서 목적
본 문서는 검색 서비스 화면의 UI 조정, 안내문 정리, 연도별 안내 로직 유지 사항, DOM 렌더링 보안 반영 사항을 외주 개발사에 전달하기 위한 변경 이력서임

## 변경 이력

### 2026-05-17 18:00

#### 1. 검색창 입력 필드 UI 개선
- 대상: `.search-input`
- 변경 내용: 입력창 테두리를 단색에서 그라데이션 스타일로 변경함
- 구현 방식: `border: 2px solid transparent;`를 적용하고, `background` 다중 레이어로 내부 흰색 배경과 외곽 그라데이션 테두리를 분리함
- 의도: 검색창을 메인 입력 요소로 강조하고, 전체 화면 톤에 맞는 완성도를 확보하기 위함임

#### 2. 검색창 내부 아이콘 적용
- 대상: `.search-input`
- 변경 내용: 돋보기 아이콘을 인라인 SVG 방식으로 표시함
- 구현 방식: `data:image/svg+xml` 기반의 SVG를 배경 레이어로 삽입하고, 좌측 패딩을 조정함
- 의도: 별도 이미지 파일 없이 CSS만으로 아이콘과 입력창을 함께 관리하기 위함임

#### 3. 검색창 비율 조정
- 대상: `.search-input`
- 변경 내용: 입력창 높이, 내부 여백, 모서리 반경을 조정함
- 구현 방식: `height`, `padding`, `border-radius` 값을 재설정함
- 의도: 기존 입력창이 화면에서 너무 두껍게 보이던 문제를 완화하고, 캡처 화면 기준 비율을 맞추기 위함임

#### 4. 검색창 하단 안내문 정리
- 대상: `.search-hint`
- 변경 내용: 예시 문구와 보조 설명의 글자 크기 및 간격을 줄임
- 구현 방식: `font-size`, `margin-top`, `line-height`를 축소함
- 의도: 입력창보다 안내문이 과도하게 강조되지 않도록 시각적 위계를 정리하기 위함임

#### 5. 학년도 탭 버튼 정리
- 대상: `.year-tab`
- 변경 내용: 탭 버튼의 패딩과 폰트 크기를 소폭 조정함
- 구현 방식: `padding`, `font-size`, `line-height` 값을 축소함
- 의도: 상단 탭이 화면에서 너무 두드러지지 않도록 조정하고, 하단 안내 박스와의 균형을 맞추기 위함임

#### 6. 미반영 안내 박스 축소
- 대상: `#emptyMessage`
- 변경 내용: 안내 박스의 여백과 글자 크기를 축소함
- 구현 방식: `padding`, `font-size`, `color` 값을 조정함
- 의도: “11월 중 반영 예정” 문구가 화면에서 지나치게 크게 보이지 않도록 하기 위함임

#### 7. 연도별 미반영 문구 자동화
- 대상: `switchYear(year)`
- 변경 내용: 선택된 학년도 값을 기준으로 안내 문구가 자동으로 갱신되도록 유지함
- 구현 방식: `year + '학년도 통학구역 정보는 11월 중 반영 예정입니다.'` 형식으로 출력함
- 의도: 연도 전환 시 수동 수정 없이 안내 문구가 일치하도록 하기 위함임

#### 8. 공공기관 안내문 정리
- 대상: 안내문 영역 및 보조 안내문
- 변경 내용: 통·반 정보, 도로명주소, 원본 학구자료, 학구도 안내 문구를 정리함
- 구현 방식: 각 항목에 대해 외부 링크 표기(`↗`)를 유지하고, 안내 문장을 공공기관 톤에 맞게 정리함
- 의도: 사용자가 참고처와 출처를 즉시 이해할 수 있도록 하기 위함임

#### 9. 학구도 안내 문구 완화
- 대상: 보조 안내문
- 변경 내용: 학구도 최신성에 대한 단정 표현을 완곡하게 정리함
- 구현 방식: `※ 현행 학구 정보가 지도에 늦게 반영될 수 있습니다.`로 표기함
- 의도: 최신 반영 여부를 단정하지 않고, 참고용 성격을 명확히 하기 위함임

#### 10. 맞춤법 및 띄어쓰기 정리
- 대상: 안내문 전체
- 변경 내용: `이용해주세요`를 `이용해 주세요`, `참고해주세요`를 `참고해 주세요`로 수정함
- 구현 방식: 공지 문구 전반의 띄어쓰기를 정리함
- 의도: 공공 안내문 표기와 맞춤법 정확도를 확보하기 위함임

#### 11. DOM 렌더링 보안 반영
- 대상: 동적 텍스트 렌더링 구간 전체
- 변경 내용: 사용자 입력값과 동적 표시 문자열 처리 시 보안 위험이 있는 문자열 삽입 방식을 점검하고, 텍스트성 데이터는 안전한 DOM 방식으로 처리하도록 반영함
- 구현 방식: plain text 성격의 값은 HTML 파싱 없이 삽입되는 방식으로 유지하고, HTML 마크업이 필요한 구간만 제한적으로 렌더링함
- 의도: XSS 등 보안 위험을 줄이고, 외주 인수 시 보안 점검에서 문제 소지가 없도록 하기 위함임

#### 12. 이벤트 처리 및 DOM 갱신 정리
- 대상: 검색창 입력, 자동완성, 결과 렌더링 구간
- 변경 내용: 중복 이벤트 바인딩을 정리하고, 검색창, 드롭다운, 결과 영역 갱신 흐름을 단일 경로로 유지함
- 구현 방식: 입력 이벤트, 키보드 탐색, 자동완성 선택, 검색 결과 갱신 로직을 분리하되 서로 중복 호출되지 않도록 정리함
- 의도: 유지보수성을 높이고, 동적 UI 갱신 시 불필요한 중복 렌더링을 줄이기 위함임

#### 13. 동적 결과 렌더링 정리
- 대상: 검색 결과 카드, 주소 블록, 태그 영역
- 변경 내용: 결과 카드 구성 요소를 역할별로 분리해 렌더링하도록 정리함
- 구현 방식: 초등학교명, 중학교/중학구, 주소 블록, 공동학구 태그, 참고 태그를 분리하여 출력함
- 의도: 외주 개발사가 추후 수정 시 어느 항목이 어느 화면 요소에 대응하는지 쉽게 파악할 수 있도록 하기 위함임

#### 14. 관리자 화면 제거 정리
대상: 관리자 화면 영역, 관리자용 CSS, 관리자용 함수 및 업로드 이벤트
변경 내용: 관리자 화면을 완전히 제거하고 사용자 검색 화면만 남도록 정리함
구현 방식: #adminView와 관련 스타일을 삭제하고, launchService, returnToAdmin, processParsedData, 파일 업로드 이벤트를 제거함
의도: 외주 개발사가 추후 유지보수 시 사용자용 화면과 관리자용 화면을 분리 없이 단순하게 이해할 수 있도록 하기 위함임

#### 15. 로드 실패 로그 강화
대상: CSV 자동 로드 예외 처리 구간
변경 내용: 파일 로드 실패 시 console.error()를 통해 원인 확인이 가능하도록 로그를 명시함
구현 방식: window.onload 내부 catch (error)에서 연도별 CSV 로드 실패 메시지를 콘솔에 출력하도록 유지/정리함
의도: 운영 화면에는 영향 없이, 배포 후 데이터 로드 문제를 빠르게 추적하기 위함임

#### 16. 검색창 접근성 라벨 추가
대상: searchInput 입력창 주변 HTML
변경 내용: 검색 입력창에 label을 연결하여 입력 목적을 명확히 표시함
구현 방식: <label for="searchInput" class="sr-only">통학구역 검색</label>를 searchInput 앞에 추가함
의도: 화면에는 보이지 않더라도 스크린리더가 검색창의 역할을 읽을 수 있도록 하여 접근성을 보강하기 위함임

#### 17. 화면비표시 라벨 스타일 추가
대상: 공통 CSS
변경 내용: sr-only 유틸리티 클래스를 추가해 라벨을 시각적으로 숨김 처리함
구현 방식: position, clip, overflow, width/height를 이용한 숨김 처리 스타일을 정의함
의도: 시각적으로는 기존 UI를 유지하면서도 보조기기에는 라벨 정보가 전달되도록 하기 위함임

## 확인 사항
- 검색창 그라데이션 테두리와 내부 배경이 분리되어 보이는지 확인함
- 돋보기 아이콘 위치가 입력 텍스트와 겹치지 않는지 확인함
- 학년도 탭 선택 시 안내 문구가 선택 연도와 일치하는지 확인함
- 안내 박스의 글자 크기와 여백이 탭보다 과도하게 크지 않은지 확인함
- 공공기관 외부 링크가 새 창으로 열리고, 링크 표기가 유지되는지 확인함
- 동적 렌더링 구간에서 plain text 입력이 HTML로 오인되지 않도록 처리되어 있는지 확인함
- 검색창에 label + sr-only를 추가한 뒤에도 시각적으로는 기존 UI가 그대로 유지되고, 스크린리더용 이름이 정상적으로 연결되는지 확인함
- CSV 로드 실패 시 console.error()가 콘솔에 출력되어 배포 후 오류 추적이 가능한지 확인함
